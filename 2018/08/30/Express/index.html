API

  • express

    • express()

      返回一个服务器对象实例

    • express.static(dirname)

      设置express服务器静态目录

    • http.createServer(app)

      http 代理express服务器

  • app

    • app.locals

      保存本地变量

    • app.delete()

      删除 路由, 中间件

    • app.use(path, fn)

      使用中间件

    • app.all(‘*’, fn)

      所有请求都经过此方法

    • app.get(path, fn), app.post(), app.put(), app.delete …

      响应不同请求方法

    • app.set(key, val)

      设置express实例变量

    • app.listen(port, [callback])

      监听端口

    • app.engine(ext, callback)

      设置模版引擎

    • app.path()

      返回express实例所对应的路径

  • request

    • req.app

      对服务器实例app的引用

    • req.ip

      请求的IP地址

    • req.url

      请求的路径

    • req.files

      请求上传的文件

    • req.method

      请求的方法

    • req.headers

      请求头

    • req.params

      请求的参数对象

    • req.query

      请求的查询参数

    • req.cookies

    • req.signedCookies

      加密的cookie 值

    • req.protocol

      返回请求的协议

    • req.param(‘key’)

      获取 get, post 请求参数

    • req.is(type)

      判断请求类型

    • req.get(headerName)

      获取请求头信息

    • req.accepts(contentType)

      根据请求头 accepts 字段判断 浏览器可以接受的文档类型

    • req.acceptsLanguage(lang), req.acceptsCharsets(charset), req.acceptsEncoding(encoding)

      判段浏览器可以接受的语言, 字符集, 编码类型

  • response

    • res.writeHead(statusCode, options)

      设置http状态吗和响应头信息

    • res.setHeader(key, value)

      设置响应头

    • res.redirect([statusCode], url)

      重定向

    • res.sendFile(filePath)

      发送文件

    • res.render(templateName, message)

      用模版渲染页面

    • res.status(code)

      返回状态码

    • res.send(body)

      发送响应体

    • res.end(body)

      发送响应并结束响应

    • res.download(path, [filename], [fn])

      下载文件

    • res.cookie(name, value, [opt])

      设置cookie

    • res.clearCookie(name, [opt])

      清除cookie

    • res.format()

      格式化输出返回, 配合req.accepts() to select handler for the request

    • res.type()

      控制返回类型

    • res.json()

      返回对象

express middleware

http://www.expressjs.com.cn/resources/middleware.html

  • 内置中间件

    • express.static()

      设置静态目录, 存放静态资源

  • 第三方中间件

    • cookie-parser

      https://github.com/expressjs/cookie-parser

      用于解析cookie到, cookie-parser不会对出站cookie做设置, 需要手动调用res.setHeader(‘Set-Cookie’: ‘…’)

      signedCookie : cookie 在出站时生成一段防篡改验证码

    • body-parser

      https://github.com/expressjs/body-parser

      处理post请求体

    • cookie-session

      https://github.com/expressjs/cookie-session

      session 作为存储在服务器端的标识符, 常常存在于散列中, 通过客户端发送的sessionid来判断会话属于哪个用户

    • compression

      https://github.com/expressjs/compression

      用来压缩响应

    • error-handler

      https://github.com/expressjs/errorhandler

      只在开发环境使用的错误处理中间件, 用于把错误信息返回给客户端

    • express-session

      https://github.com/expressjs/session

      创建session

    • morgan

      https://github.com/expressjs/morgan

      http日志请求中间件

    • passport

      https://github.com/jaredhanson/passport

      登录验证

    • multer

      https://github.com/expressjs/multer

      上传文件

      upload.html

    • serve-favicon

      https://www.npmjs.com/package/serve-favicon

      处理浏览器请求站点小图标

    • cors

      https://github.com/expressjs/cors

      处理跨域请求

    • method-override

      https://github.com/expressjs/method-override

      支持你在没有put delete的浏览器上使用这些方法

    • http-errors

      https://github.com/jshttp/http-errors

      创建错误的http响应
